# 보고서 생성 및 PUSH 워크플로우

사용자가 **"제네시스 하자"**라고 요청하면, 이 워크플로우에 따라 현재 폴더의 논의 내용을 바탕으로 보고서를 생성하고, **황원장(사용자)**의 최종 승인을 거쳐 GitHub 저장소의 `마켓인사이트` 폴더로 PUSH하는 과정을 진행합니다.


## 단계별 절차

### 1. 현 논의 내용 요약 및 보고서 생성
- 현재 폴더(`마켓인사이트`)에서 진행된 모든 논의 데이터와 시장 분석 내용을 종합합니다.
- `report.md` (또는 적절한 파일명)로 보고서를 생성합니다.

### 2. 황원장(사용자) 보고서 검토
- 생성된 보고서 파일을 황원장에게 전달하여 내용을 확인받습니다.
- **필독:** 보고서의 정확성과 완성도를 검토한 후, 실제 GitHub PUSH를 진행할지 여부는 오직 **황원장**의 결정에 따릅니다.

### 3. GitHub 저장소 PUSH
- 황원장의 최종 승인이 떨어진 경우에만 아래 주소의 `마켓인사이트` 하위 폴더로 파일을 업로드합니다.
- **URL:** `https://github.com/pwman111-debuge/stockanalysis`
- **대상 폴더:** 저장소 내 `마켓인사이트/`

## 주의 사항
- PUSH 전 반드시 `git status`를 통해 변경 사항을 확인하십시오.
- 황원장의 명시적인 승인 메시지가 있기 전까지는 어떠한 원격 작업도 수행하지 않습니다.
