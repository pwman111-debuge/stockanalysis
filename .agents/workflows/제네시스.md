# 🚀 제네시스(Genesis) 시황 분석 워크플로우

사용자가 현재 폴더에서 **"제네시스"**라고 명령하면 아래의 시황 분석 프로세스를 시작합니다.

## 1. 데이터 수집 단계 (Data Sourcing)
AI 에이전트는 다음 사이트를 검색하여 실시간 데이터를 수집합니다:
* **네이버 증권(finance.naver.com)**: 지수(코스피/코스닥), 수급 상황(외인/기관), 업종별 등락, 주요 뉴스 스캔.
* **토스 증권(tossinvest.com)**: 실시간 테마 수익률, 투자자 관심 종목, 급등 테마 분석.
* **구글 파이낸스(finance.google.com)**: 원/달러 환율, 미 증시 마감 지수, 글로벌 시장 연결성 분석.
* **기타 검증 사이트**: 인베스팅닷컴(원자재, 금리), CNN(Fear & Greed Index) 등.
※ 필요시 브라우저 캡처 및 OCR을 활용한 이미지 분석을 병행합니다.

## 2. 3단계 레이어 분석 (3-Layer Analysis)
수집된 데이터를 바탕으로 입체적 분석을 수행합니다:
* **[Layer 1] 거시 분석 (Macro)**: 금리, 환율, 글로벌 거시 지표 기반 시장 에너지 판정.
* **[Layer 2] 중기 분석 (Mid-term)**: 지수 기술적 위치, 외국인 수급 지속성, 섹터 로테이션 패턴.
* **[Layer 3] 단기 분석 (Short-term)**: 당일 실시간 수급 쏠림, 주도 테마, 변동성 요인 포착.

## 3. 시황 분석 보고서 생성 (MDX Format)
* 분석 결과는 **MDX 파일** 형식으로 작성됩니다.
* 파일명 규칙: `YYYY-MM-DD-market-analysis-genesis.mdx`
* 포함 내용: [확인 내용], [리스크 판정], [오늘의 매매 전략], [종합 투자 액션 플랜 (Do & Don't)]

## 4. 배포 제어 (Deployment)
* 작성된 MDX 보고서의 **GitHub Push 여부**는 사용자가 최종적으로 결정합니다.
* AI는 보고서 작성 완료 후 사용자에게 승인 요청을 보냅니다.
